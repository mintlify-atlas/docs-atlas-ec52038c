## Overview

The CoreDump Public Stats API provides programmatic access to coding statistics in JSON format. Use this API to integrate your coding stats into portfolio websites, personal dashboards, or any custom application.

## Endpoint

```
GET https://coredump.vercel.app/api/stats
```

This endpoint returns coding statistics for a specific user in JSON format.

## Authentication

No authentication is required. The API is publicly accessible and returns data for any CoreDump user.

## Request Parameters

<ParamField query="username" type="string" required>
  The GitHub username of the CoreDump user whose stats you want to retrieve.
</ParamField>

## Response Format

The API returns a JSON object with the following structure:

```json
{
  "username": "johndoe",
  "weekTotal": "7h 30m",
  "todayTotal": "2h 15m"
}
```

### Response Fields

<ResponseField name="username" type="string">
  The GitHub username of the user
</ResponseField>

<ResponseField name="weekTotal" type="string">
  Total coding time in the last 7 days, formatted as "Xh Ym" (e.g., "12h 45m")
</ResponseField>

<ResponseField name="todayTotal" type="string">
  Total coding time in the last 24 hours, formatted as "Xh Ym" (e.g., "3h 20m")
</ResponseField>

```typescript TypeScript
interface CoreDumpStats {
  username: string;
  weekTotal: string;
  todayTotal: string;
}

async function getStats(username: string): Promise<CoreDumpStats> {
  const response = await fetch(
    `https://coredump.vercel.app/api/stats?username=${username}`
  );
  
  if (!response.ok) {
    throw new Error('Failed to fetch stats');
  }
  
  return response.json();

const stats = await getStats('johndoe');
```

## Error Responses

The API returns appropriate HTTP status codes and error messages:

### 400 Bad Request

```json
{
  "error": "Username is required"
}
```

Returned when the `username` parameter is missing.

### 404 Not Found

```json
{
  "error": "User not found"
}
```

Returned when the specified username doesn't exist in CoreDump.

### 500 Internal Server Error

```json
{
  "error": "Internal Server Error"
}
```

Returned when an unexpected server error occurs.

## Use Cases

### Portfolio Website

Display your live coding stats on your personal website:

```jsx React Example
import { useEffect, useState } from 'react';

function CodingStats() {
  const [stats, setStats] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch('https://coredump.vercel.app/api/stats?username=johndoe')
      .then(res => res.json())
      .then(data => {
        setStats(data);
        setLoading(false);
      });
  }, []);

  if (loading) return <div>Loading stats...</div>;

  return (
    <div className="stats-card">
      <h3>My Coding Activity</h3>
      <p>This week: {stats.weekTotal}</p>
      <p>Today: {stats.todayTotal}</p>
    </div>
  );
}
```

### Dashboard Integration

Build a custom dashboard to track multiple developers:

```javascript
const developers = ['alice', 'bob', 'charlie'];

const statsPromises = developers.map(username =>
  fetch(`https://coredump.vercel.app/api/stats?username=${username}`)
    .then(res => res.json())
);

const allStats = await Promise.all(statsPromises);

allStats.forEach(stat => {
  console.log(`${stat.username}: ${stat.weekTotal} this week`);
});
```

### Mobile App

Integrate coding stats into your mobile application:

```swift Swift Example
struct CodingStats: Codable {
    let username: String
    let weekTotal: String
    let todayTotal: String
}

func fetchStats(username: String) async throws -> CodingStats {
    let url = URL(string: "https://coredump.vercel.app/api/stats?username=\(username)")!
    let (data, _) = try await URLSession.shared.data(from: url)
    return try JSONDecoder().decode(CodingStats.self, from: data)
}
```

### Discord Bot

Create a Discord bot that displays coding stats:

```javascript
const { Client } = require('discord.js');
const fetch = require('node-fetch');

client.on('messageCreate', async message => {
  if (message.content.startsWith('!codestats')) {
    const username = message.content.split(' ')[1];
    
    const response = await fetch(
      `https://coredump.vercel.app/api/stats?username=${username}`
    );
    const stats = await response.json();
    
    message.reply(
      `📊 **${stats.username}'s Coding Stats**\n` +
      `Week: ${stats.weekTotal}\n` +
      `Today: ${stats.todayTotal}`
    );
  }
});
```

## Rate Limiting

Currently, the API does not enforce rate limiting. However, we recommend:

- Caching responses when possible
- Avoiding excessive requests (more than 1 per minute per user)
- Implementing client-side throttling for real-time applications

<Note>
  Fair usage guidelines help ensure the API remains available for all users. If you need higher request volumes, consider implementing caching strategies.
</Note>

## Data Freshness

Stats are calculated in real-time from the database:

- **Week Total**: Sum of all coding time in the last 7 days
- **Today Total**: Sum of all coding time in the last 24 hours

Data updates as soon as you save files in your VS Code editor with the CoreDump extension active.

## Best Practices

### Error Handling

Always implement proper error handling:

```javascript
async function getSafeStats(username) {
  try {
    const response = await fetch(
      `https://coredump.vercel.app/api/stats?username=${username}`
    );
    
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }
    
    return await response.json();
  } catch (error) {
    console.error('Failed to fetch stats:', error);
    return null;
  }
}
```

### Caching

Implement caching to reduce API calls:

```javascript
const CACHE_DURATION = 5 * 60 * 1000; // 5 minutes
const cache = new Map();

async function getCachedStats(username) {
  const cached = cache.get(username);
  
  if (cached && Date.now() - cached.timestamp < CACHE_DURATION) {
    return cached.data;
  }
  
  const stats = await fetch(
    `https://coredump.vercel.app/api/stats?username=${username}`
  ).then(res => res.json());
  
  cache.set(username, {
    data: stats,
    timestamp: Date.now()
  });
  
  return stats;
}
```

### Type Safety

Use TypeScript interfaces for type safety:

```typescript
interface CoreDumpStats {
  username: string;
  weekTotal: string;
  todayTotal: string;
}

interface StatsError {
  error: string;
}

type StatsResponse = CoreDumpStats | StatsError;

function isError(response: StatsResponse): response is StatsError {
  return 'error' in response;
}
```

## Privacy Considerations

The Public Stats API only exposes:
- Public GitHub username
- Aggregated coding time (no file names, project details, or code)
- No personal information or email addresses

All users who sign up for CoreDump consent to having their coding time statistics publicly available.
