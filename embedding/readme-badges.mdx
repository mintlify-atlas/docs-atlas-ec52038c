## Overview

You can showcase your CoreDump coding statistics on your GitHub profile README by using the Public Stats API to fetch your data and display it in various ways.

## Using the Public Stats API

CoreDump provides a public JSON API that returns your coding statistics. You can use this data to create custom displays on your GitHub profile.

### Endpoint

```
GET https://coredump.vercel.app/api/stats?username={your-github-username}
```

### Response Format

```json
{
  "username": "johndoe",
  "weekTotal": "12h 30m",
  "todayTotal": "2h 15m"
}
```

## Display Options

### Option 1: Link to Your Profile

The simplest way to showcase your stats is to link directly to your CoreDump profile:

```markdown
## My Coding Stats

Check out my coding activity on [CoreDump](https://coredump.vercel.app/developer/yourusername)!
```

### Option 2: Use a Custom Badge Service

You can use third-party badge services like [Shields.io](https://shields.io) to create badges that fetch data from the CoreDump API:

```markdown
![Coding Time](https://img.shields.io/badge/dynamic/json?url=https://coredump.vercel.app/api/stats?username=yourusername&query=weekTotal&label=This%20Week&color=7c3aed)
```

<Info>
  Replace `yourusername` with your actual GitHub username registered on CoreDump.
</Info>

### Option 3: Create Custom Visualizations

For more advanced displays, you can:

1. Use GitHub Actions to fetch your stats periodically
2. Generate custom markdown or images
3. Commit them to your profile README repository

<Accordion title="Example GitHub Action">
  You can create a workflow that fetches your stats daily and updates your README:

  ```yaml
  name: Update Stats
  on:
    schedule:
      - cron: '0 0 * * *'  # Daily at midnight
    workflow_dispatch:
  
  jobs:
    update-stats:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v2
        - name: Fetch CoreDump Stats
          run: |
            curl https://coredump.vercel.app/api/stats?username=${{ github.repository_owner }} > stats.json
        # Add steps to update your README with the stats
  ```
</Accordion>

## What You Can Display

Using the Public Stats API, you can show:

- **Weekly coding time** - Total time spent coding in the last 7 days
- **Daily coding time** - Total time spent coding in the last 24 hours
- **Current streak** - (Available via the [Profile API](/api/profile))
- **Top languages** - (Available via the [Profile API](/api/profile))
- **Total achievements** - (Available via the [Profile API](/api/profile))
