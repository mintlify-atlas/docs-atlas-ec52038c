## Endpoint

```
GET /api/profile?username={gitUsername}
```

## Description

Retrieve detailed public profile information for a specific user by their GitHub username. This endpoint provides comprehensive statistics including:

- User profile information
- Coding streaks and achievements
- Activity breakdown by programming language
- Daily activity for the current week
- Monthly activity history

This is a public endpoint that does not require authentication.

## Authentication

No authentication required. This is a public endpoint.

## Request Parameters

<ParamField query="username" type="string" required>
  GitHub username of the user to fetch
  
  The username lookup is case-insensitive and will be normalized to lowercase.
</ParamField>

## Caching

This endpoint uses LRU caching:
- **Cache size**: 100 entries (per username)
- **TTL**: 5 minutes (300,000ms)

Cached responses include the message "Fetched from cache".

## Error Responses

### Missing Username (400)

```json
{
  "message": "Username is required",
  "status": 400
}
```

Returned when the `username` query parameter is missing.

### User Not Found (404)

```json
{
  "message": "User not found",
  "status": 404
}
```

Returned when no user exists with the specified GitHub username.

### Internal Server Error (500)

```json
{
  "message": "Something went wrong",
  "status": 500
}
```

Returned when an unexpected error occurs.

## Example Request

```bash
curl -X GET 'https://your-domain.com/api/profile?username=johndoe'
```

## Differences from Dashboard Endpoint

Unlike `/api/dashboard`, this endpoint:

- Does NOT require authentication
- Uses GitHub username instead of session
- Does NOT include private information (email, privateKey, user ID, role)
- Implements caching for better performance
- Is intended for public profile pages and widgets

## Use Cases

- Public profile pages
- Embedded coding statistics widgets
- Portfolio websites
- Social sharing of coding activity
- Third-party integrations

## Daily Activity Week Range

The `dailyActivity` array always contains 7 entries representing Monday through Sunday of the current week (week starts on Monday). Days without activity will have a duration of 0.

<Note>
  This endpoint returns public profile data only. Sensitive information like email addresses and private keys are not included.
</Note>
