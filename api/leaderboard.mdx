## Endpoint

```
GET /api/leaderboard
```

## Description

Retrieve leaderboard data for all users, including their coding statistics and activity breakdown by programming language. This endpoint is public and does not require authentication.

## Authentication

No authentication required. This is a public endpoint.

## Request Parameters

No parameters required.

## Caching

This endpoint uses LRU caching:
- **Cache size**: 100 entries
- **TTL**: 10 minutes (600,000ms)

If cached data exists, it will be returned immediately without querying the database.

## Error Responses

### Internal Server Error (500)

```json
{
  "message": "Something went wrong",
  "status": 500
}
```

Returned when an unexpected error occurs while fetching data.

## Example Request

```bash
curl -X GET https://your-domain.com/api/leaderboard
```

## Use Cases

This endpoint is ideal for:

- Displaying a global leaderboard sorted by total time, streak, or other metrics
- Comparing coding activity across different users
- Building competitive features and rankings
- Analyzing language popularity across all users

## Client-Side Sorting

The data is not pre-sorted. You can sort by various metrics on the client side:

```javascript
// Sort by total coding time
const byTotalTime = data.sort((a, b) => {
  const aTotal = a.activities.reduce((sum, act) => sum + act.totalDuration, 0);
  const bTotal = b.activities.reduce((sum, act) => sum + act.totalDuration, 0);
  return bTotal - aTotal;
});

// Sort by current streak
const byStreak = data.sort((a, b) => b.streak - a.streak);

// Sort by last 7 days activity
const byWeeklyActivity = data.sort((a, b) => {
  const aWeekly = a.activities.reduce((sum, act) => sum + act.last7DaysDuration, 0);
  const bWeekly = b.activities.reduce((sum, act) => sum + act.last7DaysDuration, 0);
  return bWeekly - aWeekly;
});
```

<Note>
  This endpoint returns all users in the database. For large user bases, consider implementing pagination or filtering on the server side.
</Note>
